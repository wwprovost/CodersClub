So far, our procedures can take input from the caller, but their only output is by printing results. It can be very useful to return a result to the caller ...

A procedure that can return a value is often known as a function. NCL has functions, but we still use the keyword procedure; the only real difference is that a procedure can use the return statement to return a computed value or result.

A procedure that can return a value is often known as a function. In Python, procedures are just functions that don't return anything; and a function can use the return statement to return a computed value or result.

A procedure that can return a value is often known as a function. In JavaScript, procedures are just functions that don't return anything; and a function can use the return statement to return a computed value or result.

You can call a function and expect the call itself to have a value. So you can set a variable to the results of the function call, and use it in the calling logic.

procedure combinations(howMany) { number oneLess = howMany - 1; return howMany * oneLess / 2; } print line "There are " + call combinations(5) + " different combinations of 5 things."; if 100 > call combinations(8) { print line "TRUE"; }
def combinations(howMany): oneLess = howMany - 1 return howMany * oneLess / 2 print "There are", combinations(5) print " different combinations of 5 things." if 100 > combinations(8): print "TRUE"
function combinations(howMany) { let oneLess = howMany - 1; return howMany * oneLess / 2; } print("There are " + combinations(5) + " different combinations of 5 things."); if (100 > combinations(8)) { print("TRUE"); }
